Windows文件传到Linux后去除^M

在Windows环境下编辑的文件，上传到Linux后会在文件中产生 ^M字符。尤其是对sh脚本的运行产生影响：

WSL下的执行结果：

原因

  CRLF(\r\n)是Windows环境下的行尾结束符，LF(\n)Linux环境下的行尾结束符，在Linux环境下，无法识别Windows环境下文件的行尾结束符引起。

处理

查看方法：cat -A <filename>

消除方法（4种）：

  • sed -i 's/^M$//g' <filename>
  • cat <filename1> | tr -d '\r' > <filename>
  • dos2unix <filename>
  • vim <filename> + 输入 : set ff=unix + 输入 :wq 退出
